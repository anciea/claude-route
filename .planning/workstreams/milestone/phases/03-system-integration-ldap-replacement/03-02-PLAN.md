**Actually, re-evaluate:** Looking at the callback endpoint, it returns JSON. For a frontend redirect flow the standard approach is:
- Backend callback redirects to frontend URL with token as query param.
- Frontend picks up the token from URL and stores it.

So modify `src/routes/authRoutes.js` callback handler (GET /auth/google/callback):
- After successfully building the response object, instead of `res.json(response)`, redirect to the SPA:
```javascript
const frontendBaseUrl = config.googleOAuth.frontendCallbackUrl || '/user-login'
const params = new URLSearchParams({
  token: sessionToken,
  user: JSON.stringify({
    id: user.id, username: user.username, email: user.email,
    displayName: user.displayName, firstName: user.firstName,
    lastName: user.lastName, role: user.role, picture: profile.picture
  }),
  isNewUser: String(isNewUser)
})
if (apiKey) params.set('apiKey', apiKey)
if (apiKeyWarning) params.set('apiKeyWarning', apiKeyWarning)
res.redirect(`${frontendBaseUrl}?${params.toString()}`)
```
- Keep the error responses as JSON (400, 401, 403, 500) -- those are for API error display, not user-facing.
