    - generateAuthUrl(): includes a cryptographically random state parameter
    - exchangeCodeForTokens(code): POST to https://oauth2.googleapis.com/token with correct grant_type, code, redirect_uri, client_id, client_secret; returns { accessToken, idToken, refreshToken, expiresAt }

    - validateDomain(email): returns true when allowedDomains is empty (all domains allowed)
    - validateDomain(email): returns true when email domain is in allowedDomains list
    - validateDomain(email): returns false when email domain is NOT in allowedDomains list
  </behavior>
  <action>
Create `src/services/googleOAuthService.js` as a class-based service (following the pattern of ldapService.js and other services in the codebase). The service should:

1. Import dependencies: `crypto` (for state generation), `axios` (already in project), `logger` (from utils), `config` (from config).

2. Define Google OAuth2 constants:
   - AUTH_URL: 'https://accounts.google.com/o/oauth2/v2/auth'
   - TOKEN_URL: 'https://oauth2.googleapis.com/token'
   - USERINFO_URL: 'https://www.googleapis.com/oauth2/v2/userinfo'
   - TOKEN_INFO_URL: 'https://oauth2.googleapis.com/tokeninfo'

3. Create `GoogleOAuthService` class with methods:

   a. `constructor()` — reads config.googleOAuth, validates required fields when enabled (clientId, clientSecret must not be empty), logs status.

   b. `generateState()` — returns `crypto.randomBytes(32).toString('base64url')` (same pattern as oauthHelper.js).

   c. `generateAuthUrl()` — builds authorization URL with URLSearchParams: client_id, redirect_uri (from config callbackUrl), response_type='code', scope (joined from config scopes array with spaces), state (from generateState()), access_type='offline', prompt='consent'. Returns `{ authUrl, state }`.

   d. `async exchangeCodeForTokens(code)` — POST to TOKEN_URL with JSON body: `{ grant_type: 'authorization_code', code, redirect_uri: config callbackUrl, client_id, client_secret }`. Uses axios with 30s timeout. Returns `{ accessToken: data.access_token, idToken: data.id_token, refreshToken: data.refresh_token || '', expiresAt: Date.now() + (data.expires_in * 1000) }`. Error handling follows existing oauthHelper.js pattern (check error.response, error.request, else).

   e. `async getUserProfile(accessToken)` — GET to USERINFO_URL with `Authorization: Bearer ${accessToken}` header. Returns `{ googleId: data.id, email: data.email, name: data.name, givenName: data.given_name || '', familyName: data.family_name || '', picture: data.picture || '', emailVerified: data.verified_email || false }`. Same error handling pattern.

   f. `async validateToken(idToken)` — GET to `${TOKEN_INFO_URL}?id_token=${idToken}`. Validates `aud` matches config clientId. Returns decoded claims. Throws on mismatch or HTTP error.

   g. `validateDomain(email)` — if `config.googleOAuth.allowedDomains` is empty array, return true. Otherwise extract domain from email and check if it's in the allowed list. Return boolean.

4. Export singleton: `module.exports = new GoogleOAuthService()`
